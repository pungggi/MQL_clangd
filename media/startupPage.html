Introducing MQL Debugging Pre-Release

We are thrilled to introduce a highly requested feature: Integrated Debugging for MQL!

Why Pre-Release? This is released early to gather your feedback. It is fully functional but undergoing active testing. If it proves valuable and stable, we'll integrate it permanently. Try it out and let us know what you think!

How it Works

Unlike traditional debuggers, this solution uses a custom Debug Bridge. When you start debugging, the extension automatically instruments your MQL code with lightweight diagnostic macros, compiles it, and launches it in MetaTrader. It then communicates with VS Code via lightning-fast file I/O to simulate a real debugging session!

Step-by-Step Guide

  • Set Breakpoints
    Open any .mq4, .mq5, or .mqh file and set breakpoints by clicking in the editor's left gutter.
  • Start Debugging
    Click the Start Debugging (bug icon) button in the editor's top title bar, or press Ctrl+Shift+D. The extension will instrument and compile your code into a special debug build (named *.mql_dbg_build).
  • Attach to Chart
    MetaTrader will launch automatically. Find your compiled debug Expert Advisor or Script in the Navigator panel and drag it onto a chart to begin execution.
  • Inspect the State
    When a breakpoint is hit, VS Code will pause. Explore your variables in the Run and Debug panel and view your Call Stack.
  • Resume Execution
    Use the VS Code debug toolbar to click Continue and let your code run until the next breakpoint.
  • Stop Debugging
    When you are finished, click Stop in VS Code. Remember to manually remove the debug EA/Script from your MetaTrader chart so it stops running!

What Variables are Tracked?

To keep performance extremely high, the debugger smartly tracks variables based on your settings:

  • Locals & Parameters: Tracks local variables and function parameters accessed near your breakpoints.
  • Class Members: Monitors fields belonging to the current class instance (this).
  • Deep Analysis (Optional): Want more details? Enable mql_tools.Debug.DetailLevel = "deepAnalysis" in your VS Code settings. This will additionally track global primitives and recursively expand object fields.
  • Breakpoint Info: Provides metadata such as hit counts, source file names, line numbers, and timestamps.

Advanced Features

  • Conditional Breakpoints: Right-click a breakpoint and add a condition — telemetry only fires when the condition is true.
  • Manual // @watch annotations: Add a // @watch myVar otherVar comment near a breakpoint to explicitly name variables the auto-detector might miss.
  • Call Stack Tracking: Functions containing breakpoints automatically report ENTER/EXIT events so the dashboard shows a live call stack.
⚠ Pause / Continue: Each breakpoint injects a blocking spin-wait. The EA thread is fully paused — no OnTick/OnTimer events fire. Use only on demo accounts or in the Strategy Tester. The EA auto-resumes after 120 seconds as a safety failsafe.

Trade Report Dashboard Pre-Release

Analyze your Strategy Tester results directly in VS Code — no more digging through MetaTrader's UI or raw log files.

The Trade Report Dashboard parses MT5 tester log files and presents trades, P&L statistics, and log entries in a rich interactive webview — with click-to-source navigation back to your MQL code.

Opening a Report

Run MQL: Open Trade Report Dashboard from the Command Palette. The dashboard auto-discovers EAs that have tester log files under your MQL5/Experts/ folder.

What the Dashboard Shows

SectionDetails
Trade SummaryTotal trades, net P&L, win rate, gross profit & loss, commissions
Trade TableEntry/exit prices, SL, TP, lot size, exit reason per trade
Log ViewerFilterable by level: ALL / TRADE / INFO / WARN / DEBUG / ERROR

Click-to-Source Navigation

When your EA uses LiveLog's level-prefixed logging macros, each log entry and trade automatically carries a source location. In the dashboard these appear as clickable badges that jump straight to the originating line in your MQL file.

Just add to your EA:

#include <LiveLog.mqh>

void OnTick()
{
    LogInfo("Checking for entry signal");
    if (buySignal)
        LogTrade("SIMULATED BUY MARKET");
}

The LogDebug, LogInfo, LogWarn, LogError, and LogTrade macros embed {File:Function:Line} tags automatically — no extra setup needed.

Source Snapshots

Because source tags reference specific line numbers, editing your EA after a test run can make links point to the wrong lines. Enable Source Snapshots to freeze a copy of all referenced files alongside each log:

"mql_tools.TradeReport.SnapshotSources": true

When a snapshot exists, each source badge shows two links: a green one for the frozen snapshot and a yellow one for the current live file.

Run Backtest Pre-Release

Trigger an MT5 Strategy Tester run for your EA directly from VS Code — without touching the MetaTrader UI.

Press Ctrl+Shift+T, click the ⚗ Run Backtest toolbar button, or run MQL: Run Backtest from the Command Palette. When the test finishes, the Trade Report Dashboard opens automatically.

Prerequisites

  • A tester.ini file must exist in your EA's folder (e.g. Experts/MyEA/tester.ini). MetaTrader generates this when you save Strategy Tester settings.
  • TradeReportServer must be running. By default the extension auto-starts it — see mql_tools.Backtest.AutoStartServer.

Step-by-Step Guide

  • Open your EA file
    Open any .mq5, .mq4, or .mqh file that belongs to your Expert Advisor.
  • Start the backtest
    Press Ctrl+Shift+T or click the ⚗ button in the editor title bar. The extension auto-detects your EA from the active file. If multiple EAs are found, a picker appears.
  • Choose parameters
    A prompt pre-filled from tester.ini lets you confirm or override the symbol and date range. Disable prompts entirely with mql_tools.Backtest.PromptForParameters: false.
  • Monitor progress
    A VS Code notification tracks elapsed time while MT5 runs the Strategy Tester in the background. Cancelling the notification stops monitoring but does not cancel the MT5 test itself.
  • View results
    When the test finishes, the Trade Report Dashboard opens automatically with the new results (disable with mql_tools.Backtest.AutoOpenReport: false).

Settings Reference

SettingDefaultDescription
mql_tools.Backtest.PromptForParameterstrueShow symbol / date prompts before running. Set false to use tester.ini defaults silently.
mql_tools.Backtest.AutoOpenReporttrueOpen the Trade Report Dashboard when the test completes.
mql_tools.Backtest.AutoStartServertrueAuto-start TradeReportServer if it is not already running.
mql_tools.Backtest.ServerPort3002Port used by TradeReportServer.
mql_tools.ShowButton.RunBacktesttrueShow / hide the ⚗ toolbar button on MQL files.

Live Runtime Log Pre-Release

Monitor your MQL4/MQL5 terminal logs in real-time directly within VS Code — no need to switch to MetaTrader or open external log files.

The Live Log feature tails your MetaTrader log files and streams new entries into a dedicated VS Code output channel as they appear. Toggle it from the status bar or with MQL: Toggle Live Runtime Log.

Two Tailing Modes

ModeHow it worksLatency
LiveLog (Real-time) Tails MQL5/Files/LiveLog.txt. Your EA writes to this file with an immediate flush using PrintLive() or the Log*() macros from LiveLog.mqh. ⚡ Instant
Standard Journal Tails the daily MQL5/Logs/YYYYMMDD.log file produced by MetaTrader's built-in Print(). Delayed (MT5 buffers output)

Switch between modes at any time with MQL: Switch Log Tail Mode (Live/Standard).

Setting Up LiveLog (Real-Time Mode)

  • Install the library
    Run MQL: Install LiveLog Library from the Command Palette. This deploys LiveLog.mqh into your MetaTrader Include/ folder.
  • Include it in your EA
    #include <LiveLog.mqh>
  • Log messages with instant flush
    Use PrintLive() anywhere you would use Print(): 
    PrintLive("OnTick fired — bid: " + DoubleToString(bid, 5));
PrintFormatLive("Signal: %d | Spread: %.1f pts", signal, spread);
  • Use level-prefixed macros for Trade Report integration
    The Log*() macros embed a {File:Function:Line} source location tag automatically, enabling click-to-source navigation in the Trade Report Dashboard: 
    LogDebug("Checking entry conditions");
LogInfo("Order placed — lots: 0.10, SL: 1.2050");
LogWarn("Spread too wide, skipping");
LogError("OrderSend failed: " + IntegerToString(GetLastError()));
LogTrade("SIMULATED BUY MARKET");
  • (Optional) Redirect all Print() calls automatically
    Add #define LIVELOG_REDIRECT before the include to make every Print() and PrintFormat() go to LiveLog automatically — no code changes needed: 
    #define LIVELOG_REDIRECT
#include <LiveLog.mqh>

// All Print() calls now flush immediately to LiveLog.txt
Print("This appears in VS Code in real-time!");
  • (Optional) Clean session end marker
    Call LiveLogClose() from your OnDeinit to write a "Session Ended" marker in the log: 
    void OnDeinit(const int reason)
{
    LiveLogClose();
}

Commands

  • MQL: Toggle Live Runtime Log — Start / stop log tailing (also available in the status bar)
  • MQL: Install LiveLog Library — Deploy LiveLog.mqh to your Include folder
  • MQL: Switch Log Tail Mode (Live/Standard) — Switch between real-time and standard modes

Notes

  • Only new log entries are shown — historical entries are not replayed when tailing starts.
  • LiveLog.txt auto-rotates at 10 MB (renamed to LiveLog_YYYY_MM_DD.txt) to prevent disk space issues.
  • The extension auto-detects your MQL version (4 or 5) and data folder from the active file and workspace structure. Set mql_tools.Metaeditor.Include5Dir or Include4Dir explicitly if auto-detection fails.
