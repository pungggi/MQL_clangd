Compile Listener Service (MQL5 Only)

Automatically reload a running EA in MetaTrader after every successful compile — without touching the terminal.

Note: This feature is strictly MQL5-only (MQL4 does not support services or the required compilation structure).

How it works: VS Code writes a flag file after compile. The MT5 Service polls for that file, finds the chart running your EA, and calls ChartApplyTemplate() to reload it.
Note: The hook triggers on all successful compile operations including CheckOnSave/auto-check (meaning it may run on automatic checks). This is entirely intentional: the flag file write is idempotent, and the reload operation runs on a microsecond scale without overhead.

Setup

  • Install and compile the service
    Click the button below — the extension copies CompileListenerService.mq5 directly into your MQL5/Services/ folder. Then compile it.
  • Save a template for your EA
    In MT5, open the chart running your EA → right-click → Templates → Save As → name the template file exactly after your EA, including the .tpl extension (e.g. MyEA.tpl). When configuring the service input inside MT5, enter the EA name strictly without any file extension (e.g., enter MyEA, not MyEA.tpl or MyEA.mq5).
  • Create the flag folder
    Run the command below in your terminal depending on your platform:
    Windows (Command Prompt / PowerShell):
    mkdir "%APPDATA%\MetaQuotes\Terminal\<ID>\MQL5\Files\COMPILEFLAGS"
    macOS / Linux (Wine):
    mkdir -p "$HOME/.wine/drive_c/users/$USER/AppData/Roaming/MetaQuotes/Terminal/<ID>/MQL5/Files/COMPILEFLAGS"
    Replace <ID> with your terminal's data folder ID (find it via File → Open Data Folder in MT5).
  • Add the VS Code task
    Create or open .vscode/tasks.json and add the appropriate configuration for your platform:
    Windows (cmd.exe variant):
    {
  "version": "2.0.0",
  "tasks": [
    {
      "label": "write-compile-flag",
      "type": "shell",
      "command": "cmd",
      "args": [
        "/c",
        "echo.>\"%APPDATA%\\MetaQuotes\\Terminal\\<ID>\\MQL5\\Files\\COMPILEFLAGS\\${fileBasenameNoExtension}.flag\""
      ],
      "presentation": { "reveal": "never" }
    }
  ]
}
    macOS / Linux / Wine (bash variant):
    {
  "version": "2.0.0",
  "tasks": [
    {
      "label": "write-compile-flag",
      "type": "shell",
      "command": "bash",
      "args": [
        "-c",
        "touch \"$HOME/.wine/drive_c/users/$USER/AppData/Roaming/MetaQuotes/Terminal/<ID>/MQL5/Files/COMPILEFLAGS/${fileBasenameNoExtension}.flag\""
      ],
      "presentation": { "reveal": "never" }
    }
  ]
}
    💡 Cross-Platform Hint: You can use VS Code environment variables like ${env:APPDATA} or ${env:HOME} within task configurations to reference path environments dynamically without hardcoding them.
  • Wire the task to compile
    Add to your VS Code settings: 
    "mql_tools.Compile.RunTaskOnSuccess": "write-compile-flag"
    💡 Behavior Note: This hook fires on all successful compile operations, including those triggered by CheckOnSave/auto-check. This behavior is intentional, as writing the flag file is fully idempotent and the reload process is microsecond-scale, causing no performance impact.
  • Start the service in MT5
    Navigator → Services → CompileListenerService → double-click → set expertNames to your EA name(s) → OK → right-click → Start.

Service Inputs

Input Default Purpose
expertNames "EA_Name_1,EA_Name_2" Comma-separated EA names to watch (no extension). Whitespace is trimmed.
showDebugOutput true Print per-chart scan info and template-apply lines to the Experts log.

Verifying it works

Compile your EA from VS Code. In the MT5 Experts log you should see:

FOUND EA MyEA on chart 123456789, applying template MyEA.tpl
Note: The flag file path is relative to MQL5/Files/ — the VS Code task must write to MQL5/Files/COMPILEFLAGS/<name>.flag. The service uses MT5's FileIsExist / FileDelete APIs which are sandboxed to that folder.

Notes

  • The service is a true #property service — it runs independently of any chart and survives terminal reload.
  • Watch multiple EAs by separating names with commas: "MyEA, OtherEA".
  • For each EA, a template named <EA>.tpl must exist in MQL5/Profiles/Templates/.
  • Failures from FileDelete and ChartApplyTemplate are always logged with GetLastError().
