Trade Report Dashboard Pre-Release

Analyze your Strategy Tester results directly in VS Code — no more digging through MetaTrader's UI or raw log files.

How to get the most out of it? The Trade Report Dashboard provides maximum value when used together with the Run Backtest feature and the LiveLog library. The dashboard parses MT5 tester log files and presents trades, P&L statistics, and log entries in an interactive webview — but paired with LiveLog, it unlocks click-to-source navigation directly back to your MQL code.

How to Use It

The Trade Report Dashboard works with any standard MetaTrader 5 backtest. You can use it in two ways:

  • Standalone (Manual): Run a normal backtest directly within MetaTrader 5's Strategy Tester. When finished, switch to VS Code and run MQL: Open Trade Report Dashboard from the Command Palette. The dashboard will automatically parse your standard MT5 tester log files into an interactive visualization.
  • Integrated (Recommended): Trigger the backtest directly from VS Code using the Run Backtest feature (Ctrl+Shift+T). When the test completes, the dashboard opens automatically with your new results.

What the Dashboard Shows

Section Details
Trade Summary Total trades, net P&L, win rate, gross profit & loss, commissions
Trade Table Entry/exit prices, SL, TP, lot size, exit reason per trade
Log Viewer Filterable by level: ALL / TRADE / INFO / WARN / DEBUG / ERROR

Click-to-Source Navigation

When your EA uses LiveLog's level-prefixed logging macros, each log entry and trade automatically carries a source location. In the dashboard these appear as clickable badges that jump straight to the originating line in your MQL file.

Just add to your EA:

#include <LiveLog.mqh>

void OnTick()
{
    LogInfo("Checking for entry signal");
    if (buySignal)
        LogTrade("SIMULATED BUY MARKET");
}

The LogDebug, LogInfo, LogWarn, LogError, and LogTrade macros embed {File:Function:Line} tags automatically — no extra setup needed.

Source Snapshots

Because source tags reference specific line numbers, editing your EA after a test run can make links point to the wrong lines. Enable Source Snapshots to freeze a copy of all referenced files alongside each log:

"mql_tools.TradeReport.SnapshotSources": true

When a snapshot exists, each source badge shows two links: a green (snapshot) one for the frozen snapshot and a yellow (live) one for the current live file.
