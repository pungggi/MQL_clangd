Live Runtime Log

Monitor your MQL4/MQL5 terminal logs in real-time directly within VS Code — no need to switch to MetaTrader or open external log files.

The Live Log feature tails your MetaTrader log files and streams new entries into a dedicated VS Code output channel as they appear. Toggle it from the status bar or with MQL: Toggle Live Runtime Log.

Two Tailing Modes

Mode How it works Latency
LiveLog (Real-time) Tails MQL5/Files/LiveLog.txt. Your EA writes to this file with an immediate flush using PrintLive() or the Log*() macros from LiveLog.mqh. ⚡ Instant
Standard Journal Tails the daily MQL5/Logs/YYYYMMDD.log file produced by MetaTrader's built-in Print(). Delayed (MT5 buffers output)

Switch between modes at any time with MQL: Switch Log Tail Mode (Live/Standard).

Setting Up LiveLog (Real-Time Mode)

  • Install the library
    Run MQL: Install LiveLog Library from the Command Palette. This deploys LiveLog.mqh into your MetaTrader Include/ folder.
  • Include it in your EA
    #include <LiveLog.mqh>
  • Log messages with instant flush
    Use PrintLive() anywhere you would use Print(): 
    PrintLive("OnTick fired — bid: " + DoubleToString(bid, 5));
PrintFormatLive("Signal: %d | Spread: %.1f pts", signal, spread);
  • Use level-prefixed macros for Trade Report integration
    The Log*() macros embed a {File:Function:Line} source location tag automatically, enabling click-to-source navigation in the Trade Report Dashboard: 
    LogDebug("Checking entry conditions");
LogInfo("Order placed — lots: 0.10, SL: 1.2050");
LogWarn("Spread too wide, skipping");
LogError("OrderSend failed: " + IntegerToString(GetLastError()));
LogTrade("SIMULATED BUY MARKET");
  • (Optional) Redirect all Print() calls automatically
    Add #define LIVELOG_REDIRECT before the include to make every Print() and PrintFormat() go to LiveLog automatically — no code changes needed: 
    #define LIVELOG_REDIRECT
#include <LiveLog.mqh>

// All Print() calls now flush immediately to LiveLog.txt
Print("This appears in VS Code in real-time!");
  • (Optional) Clean session end marker
    Call LiveLogClose() from your OnDeinit to write a "Session Ended" marker in the log: 
    void OnDeinit(const int reason)
{
    LiveLogClose();
}

Commands

  • MQL: Toggle Live Runtime Log — Start / stop log tailing (also available in the status bar)
  • MQL: Install LiveLog Library — Deploy LiveLog.mqh to your Include folder
  • MQL: Switch Log Tail Mode (Live/Standard) — Switch between real-time and standard modes

Notes

  • Only new log entries are shown — historical entries are not replayed when tailing starts.
  • LiveLog.txt auto-rotates at 10 MB (renamed to LiveLog_YYYY_MM_DD.txt) to prevent disk space issues.
  • The extension auto-detects your MQL version (4 or 5) and data folder from the active file and workspace structure. Set mql_tools.Metaeditor.Include5Dir or Include4Dir explicitly if auto-detection fails.
